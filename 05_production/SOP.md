# SOP — 生产流程（v5 · Grok 批产版）

## 角色分工
| 角色 | 职责 |
|------|------|
| **老板（人）** | 终审：基调裁决、合并放行 |
| **导演（Kimi）** | 出任务卡、逐章审稿、合并、更新账本 |
| **写作 Agent（Grok 等）** | 按任务卡写正文，不改设定 |

## 批次纪律
**每批 3–5 章**，导演逐章过质量门，过了进 `章节/`，账本同步，再放行下一批。
禁止连跑 10+ 章不送审。

## 投喂包（写作 Agent 开写前必发，导演按章裁剪）
1. `00_canon/00_CORE.md`
2. `01_world/WORLD.md`（时代红线必读）
3. `04_style/STYLE.md`
4. `02_characters/CAST.md` 本章出场人物卡（只发相关卡）
5. `06_continuity/LEDGER.md` 全文
6. 已合并正文最近 2 章（文风锚点）
7. 本章任务卡（模板见下）
8. **本章笑点检索摘要**（导演或写作 Agent 自搜，见下节；无检索摘要=不开写）

## 笑点检索协议（强制 · 防平庸）
开写前必须 **联网检索** 与本章情境相关的喜剧结构／有趣桥段，再落笔。禁止凭空堆「万能吐槽」。

**检索方向（举例，按章选题）：**
- 反差／大力出洋相／指挥背锅／睡着办事
- 客服话术／倒计时／服务到期／续费话术（给狗系统用）
- 误会链／信息差滚雪球／复读三次变味
- 神话去魅：大人物的小毛病

**转化铁律（比检索更重要）：**
1. **取结构，不取热词。** 禁止把网络流行语原句塞进古人嘴里；现代梗只许苏慕内心 + 狗系统营业腔。
2. **笑话必须长在人物身上。** 招摇=大力熊孩子，白耳=冷面拆台，鹿蜀=KPI祥瑞——检索结果要改写成他们会做的事。
3. **每章至少 1 个「记忆点笑点」**：读者合上书还能复述的那个动作／那句错位（不是三句平庸吐槽凑数）。
4. **章末自报须写：** 检索来源一句话 + 转化成了哪个原创笑点。导演审「笑点门」时对照：像模板就退稿。

## 剧情别出心裁（强制 · 防套路）
节拍可以跟任务卡，**解法与场面不许抄网文默认解**。每章至少一处让读者意外但仍自洽的设计，例如：
- 用错的知识碰巧救场／用对的知识反而惹祸
- 祥瑞办事像办业务，神异像事故现场
- 护送／对峙／谈判用「礼节／误会／流程」推进，不靠战力升级
- 钩子要具体（一个动作、一句错位），禁止空悬念

导演审稿：若本章冲突解法可换到任意修仙文仍成立 → 退稿重写场面。

## 衔接与出场（强制 · 防凭空、防捣腾）
批量并行写章最易翻车处。导演审合前必须过「衔接门」，写作 Agent 开写前必须写清承接状态。

1. **出场有因**：任何人（含兽）入场须有上一章钩子、行程、呼喊、跟随、回村等理由。禁止「下一句突然站在旁边」。CAST 未登场角色不得无预告登场。
2. **状态连续**：地点／队伍构成／伤势光晕／手中物件／倒计时，必须与 LEDGER + 上章末对齐。换地点须写「怎么到的」至少一句动作。
3. **一人一事一时**：同一章内禁止无必要地来回倒腾（刚西行又无故东返、刚入山又无因出缝再入）。案与案切换只走任务卡规定的钩子。
4. **并行写作护栏**：同批后章不得假设前章未写明的新事实；若需共用状态，以 LEDGER／导演投喂的「承接摘要」为准，不得私自发明。
5. **退稿标准**：凭空加人、时空跳跃无过渡、同一冲突反复横跳、上章钩子下落不明 → 一门不过即退。

## 任务卡模板（导演逐章填写）
```
【第 NNN 章】标题：
一、本章承接：上章结尾状态（地点/人物/悬念）
二、必达节拍（3–5 条，按序）：
三、出场人物（各附：声音、禁用句、本章动机）：
四、本章知识（≥1 条真山海经条目，注明篇目+用法）：
五、笑点配额（≥3，注明类型：反差/拆解崇高/职业病/复读梗/误会链）：
   ※其中≥1条须标「记忆点」，并附检索转化说明
六、泪点（如有，一刀即止，注明埋法）：
七、雷（埋/爆，登记编号）：
八、禁词与时代红线提醒：
九、章末钩：
十、字数：2800–4500
```

## 交付格式（写作 Agent 交稿）
- 正文 2800–4500 字。
- 章末自报：新立事实／新埋伏笔／代价记录／本章笑点清单（含记忆点+检索转化）／知识条目／钩子／禁词自检。

## 质量门（逐章对照，一门不过即退稿）
| 门 | 标准 |
|----|------|
| 设定门 | 不违背 CORE/WORLD/CAST 任一条；新事实登记 |
| 时代门 | 无越时代物件（书纸字钱官府考试客栈等，见 WORLD §3） |
| 泄密门 | 本卷禁词零出现；深层异常未超配额（WORLD §4） |
| 笑点门 | ≥3 个有效笑点且长在人物身上；**≥1 个记忆点**；无检索转化说明则退；像万能吐槽模板则退 |
| 衔接门 | 出场有因；状态与上章/LEDGER连续；无凭空加人；无无必要来回倒腾；章末钩须被下章接住 |
| 知识门 | ≥1 条真山海经条目，随剧情变现 |
| 文风门 | 符合 STYLE：短句对话驱动、内心戏≤20%、章末有钩 |
| 题眼门 | 本批至少一处呼应"人/记"题眼（只演示，可极轻） |

## 合并与账本
- 过门稿件进 `章节/第NNN章_标题.txt`。
- 导演更新 LEDGER：已锁事实、谜题开关、校勘进度、梗台账。
- 每卷末出"读者体验审计"：笑点/泪点/谜题开关账，供老板终审。
