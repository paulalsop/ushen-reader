# SOP — 生产流程（v7 · 多agent逐章交付版）

## 角色分工
| 角色 | 职责 |
|------|------|
| **老板（人）** | 终审：基调裁决、合并放行 |
| **主审 Agent** | 出任务卡、逐章审稿、更新账本与规范 |
| **写作 Agent** | 按任务卡写正文，只改授权章节，不擅改设定 |

## 并行纪律

- 可以连续推进多章，但必须**一章一审、一章一提交、一章一推送**；不得攒十章后一次性合并。
- 写作前先确认工作树与远端状态。工作树不干净时，只处理自己被授权的文件，不覆盖、不删除、不暂存其他agent的修改。
- 每章只暂存该章文件。文档更新另开独立提交，不与正文章混合。
- 推送冲突时先同步远端，再只解决自己章节的冲突；不得用重置、强推或整树覆盖清除他人工作。
- 同一章同一时刻只交给一个agent。后章不得假设前章尚未提交的新事实。

## 投喂包（写作 Agent 开写前必发，导演按章裁剪）
1. `00_canon/00_CORE.md`
2. `01_world/WORLD.md`（时代红线必读）
3. `04_style/STYLE.md`
4. `02_characters/CAST.md` 本章出场人物卡（只发相关卡）
5. `06_continuity/LEDGER.md` 全文
6. 已合并正文最近 3 章（文风与连续性锚点）
7. 本章任务卡（模板见下）

## 剧情别出心裁（强制 · 防套路）
节拍可以跟任务卡，**解法与场面不许抄网文默认解**。每章至少一处让读者意外但仍自洽的设计，例如：
- 用错的知识碰巧救场／用对的知识反而惹祸
- 祥瑞办事像办业务，神异像事故现场
- 护送／对峙／谈判用「礼节／误会／流程」推进，不靠战力升级
- 强钩要具体（一个动作、一句错位），禁止空悬念

导演审稿：若本章冲突解法可换到任意修仙文仍成立 → 退稿重写场面。

## 衔接与出场（强制 · 防凭空、防捣腾）
1. **出场有因**：任何人（含兽）入场须有上一章钩子、行程、呼喊、跟随、回村等理由。
2. **状态连续**：地点／队伍构成／伤势光晕／手中物件／倒计时，必须与 LEDGER + 上章末对齐。
3. **一人一事一时**：同一章内禁止无必要地来回倒腾。
4. **并行写作护栏**：同批后章不得假设前章未写明的新事实。
5. **退稿标准**：凭空加人、时空跳跃无过渡、同一冲突反复横跳、上章钩子下落不明 → 一门不过即退。
6. **地图门（书即行程单）**：卷一锁定南山经䧿山系西→东（见 `cards/V01.md` 行程地图）。禁止跳山/瞬移。

## 任务卡模板（导演逐章填写）
```
【第 NNN 章】标题：
一、本章承接：上章结尾状态（地点/人物/资源/目标）
二、行程：上一站 → 本章所在 → 章末指向（原文里程；回程/快进须写明）
三、戏剧问题：谁现在想得到什么？
四、人物压力：这件事咬中了他的哪种欲望、恐惧或旧债？
五、核心选择：他必须在什么与什么之间选择？
六、场景转折：哪一刻使原计划失效？（1–2 个结果，禁止列 3–5 个待办动作）
七、状态变化：章末人物/关系/资源/目标有什么不同？
八、持续后果：本章选择立即付出什么，并将在何章继续兑现？
九、线索增量：新观察改变了哪个旧判断？无则 —
十、出场人物（各附：声音、禁用句、本章动机）：
十一、知识用途：改变行动 / 造成误判 / 兑现后果；无则 —
十二、喜剧发动机：人物行为或关系如何自然产笑？无则 —
十三、泪点（如有，一刀即止）：
十四、雷（埋/爆，登记编号）：
十五、禁词与时代红线提醒：
十六、结尾方式：决定 / 后果 / 关系变化 / 意象 / 强钩
十七、字数：普通章≥3000；重要冲突章3500–4500（统计到首个---）
十八、信息边界：亲见/听说/猜测/未知；持有人；外传权限；系统声道
```

## 交付格式（写作 Agent 交稿）
- 普通章正文原则上不少于3000字；重要冲突章3500–4500字。统计到第一个 `---`，不含章末自报。
- 章末自报（供导演/账本，**不当读者正文**）：
  - 新立事实
  - 选择与不可逆后果
  - 线索已知结论（写清「读者已知什么」，禁止后章再当新发现）
  - 雷台账变动
  - 校勘进度（如有）
  - 禁词自检

## 质量门（逐章对照，一门不过即退稿）
| 门 | 标准 |
|----|------|
| 设定门 | 不违背 CORE/WORLD/CAST 任一条；新事实登记 |
| 时代门 | 无纸钱官府考试客栈等越时代物件；无成熟通用文字体系。残册与私人符号按 WORLD 边界处理 |
| 载体门 | 结绳、刻痕、骨符、泥形、脚印与实物不能自行讲完整句；须有持有人/见证人口述 |
| 声道门 | 系统私语/共听清楚；古人不回应私语；白耳与招摇不读心、不全知 |
| 泄密门 | 本卷禁词零出现；深层异常未超配额（WORLD §4） |
| 因果门 | 有即时目标、阻力意志、核心选择；章末状态≠章初 |
| 后果门 | 选择付出代价或明确将兑现；「继续东走」不算 |
| 线索门 | 新线索须改变判断或行动；只重复气氛=退 |
| 喜剧门 | 若有笑点须长在人物行动上；有解释笑点句=退；万能吐槽模板=退 |
| 衔接门 | 出场有因；状态与上章/LEDGER连续；无凭空加人 |
| 地图门 | 行程与 V01 地图一致；无跳山；已校山掠影不重案 |
| 知识门 | 若有知识须改变行动/造成误判/兑现后果；无知识允许 |
| 文风门 | 符合 STYLE：短句对话驱动、内心戏≤20%；无审计腔、提纲腔、机械“A不是B，是C” |
| 密度门 | 正文达到目标篇幅；润色未缩水；扩写均为动作、关系、空间、伏笔或后果 |
| 格式门 | 文件名、首行章号、标题一致；首个---后仅一份章末自报；无重复段和多版本结尾 |
| 题眼门 | 本批至少一处呼应「人/记」题眼（只演示，可极轻） |

### 场景验收七问（导演审稿时自问）
1. 此刻谁想要什么？
2. 阻力来自另一个人的意志，还是仅来自作者暂不公布信息？
3. 场景中谁做了选择？
4. 选择后失去了什么？
5. 删除本场，后文是否完全不受影响？如果是，删场。
6. 把苏慕换成任意聪明人，场景是否仍成立？如果是，主线没有咬住人物。
7. 笑点之后是否还有一句解释它为什么好笑？有则删解释句。

## 单元级指标（每 10–20 章一案，案末审计）
- **喜剧**：1 个人物性喜剧发动机（建立→升级→变味或回收）；2–4 个真正承担场面的喜剧段落。
- **知识**：2–4 条核心条目；至少两次影响决策、误判或代价。
- **钩子**：仅在转折点配置 2–4 个强钩；其余章可用决定/后果/关系变化收束。
- **不可逆后果**：至少一次由主角选择造成的持续损失（信任、资源、可选道路、关系、目标）。
- **线索增量链**：观察 → 假设修正 → 因假设行动 → 行动带来新风险。禁止五章内把同一答案换说法重讲。

## 合并与账本
- 过门稿件进 `章节/第NNN章_标题.txt`。
- 每章完成后执行：篇幅统计 → 禁词/越界词检查 → 相邻章衔接 → `git diff --check` → 只暂存本章 → 提交 → 同步远端 → 推送。
- 导演更新 LEDGER：已锁事实、关系状态、不可逆后果、线索已知结论、雷台账、校勘进度。
- 每案末出「读者体验审计」：重复、解释笑点、无后果、人物可替换问题；供老板终审。
- 阅读器发布时自动剔除【章末自报】；写作 Agent 仍须附自报。
