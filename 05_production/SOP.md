# SOP — 生产流程（v5 · Grok 批产版）

## 角色分工
| 角色 | 职责 |
|------|------|
| **老板（人）** | 终审：基调裁决、合并放行 |
| **导演（Kimi）** | 出任务卡、逐章审稿、合并、更新账本 |
| **写作 Agent（Grok 等）** | 按任务卡写正文，不改设定 |

## 批次纪律
**每批 3–5 章**，导演逐章过质量门，过了进 `章节/`，账本同步，再放行下一批。
禁止连跑 10+ 章不送审。

## 投喂包（写作 Agent 开写前必发，导演按章裁剪）
1. `00_canon/00_CORE.md`
2. `01_world/WORLD.md`（时代红线必读）
3. `04_style/STYLE.md`
4. `02_characters/CAST.md` 本章出场人物卡（只发相关卡）
5. `06_continuity/LEDGER.md` 全文
6. 已合并正文最近 2 章（文风锚点）
7. 本章任务卡（模板见下）

## 任务卡模板（导演逐章填写）
```
【第 NNN 章】标题：
一、本章承接：上章结尾状态（地点/人物/悬念）
二、必达节拍（3–5 条，按序）：
三、出场人物（各附：声音、禁用句、本章动机）：
四、本章知识（≥1 条真山海经条目，注明篇目+用法）：
五、笑点配额（≥3，注明类型：反差/拆解崇高/职业病/复读梗/误会链）：
六、泪点（如有，一刀即止，注明埋法）：
七、雷（埋/爆，登记编号）：
八、禁词与时代红线提醒：
九、章末钩：
十、字数：2800–4500
```

## 交付格式（写作 Agent 交稿）
- 正文 2800–4500 字。
- 章末自报：新立事实／新埋伏笔／代价记录／本章笑点清单／知识条目／钩子／禁词自检。

## 质量门（逐章对照，一门不过即退稿）
| 门 | 标准 |
|----|------|
| 设定门 | 不违背 CORE/WORLD/CAST 任一条；新事实登记 |
| 时代门 | 无越时代物件（书纸字钱官府考试客栈等，见 WORLD §3） |
| 泄密门 | 本卷禁词零出现；深层异常未超配额（WORLD §4） |
| 笑点门 | ≥3 个有效笑点，长在人物身上，不解说 |
| 知识门 | ≥1 条真山海经条目，随剧情变现 |
| 文风门 | 符合 STYLE：短句对话驱动、内心戏≤20%、章末有钩 |
| 题眼门 | 本批至少一处呼应"人/记"题眼（只演示，可极轻） |

## 合并与账本
- 过门稿件进 `章节/第NNN章_标题.txt`。
- 导演更新 LEDGER：已锁事实、谜题开关、校勘进度、梗台账。
- 每卷末出"读者体验审计"：笑点/泪点/谜题开关账，供老板终审。
